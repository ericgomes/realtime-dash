Sobre este dashboard e métricas

Mostra a performance de carregamento medida em usuários reais (RUM), coletada pelo navegador via GTM. Cada page view é um carregamento de página (evento window load). Os dados são amostrados (uma fração dos page views) e consolidados em resumos por período. A leitura é neutra: apresenta tempos e distribuições, sem rotular acessos como bons ou ruins.

É um monitor ao vivo: mantém apenas as últimas 3 horas. Page views mais antigos são descartados automaticamente — o histórico de longo prazo fica no GA4. Por isso os períodos vão só até 3h.

Métricas

TTFB (Time To First Byte)
Tempo entre a requisição e o primeiro byte da resposta. Reflete servidor e rede, antes do frontend renderizar.
DOM Ready
Tempo até a estrutura da página ficar pronta (DOMContentLoaded), antes de imagens e recursos pesados.
Load
Tempo total até a página terminar de carregar (loadEventEnd). É o tempo de "página pronta".
Média
Soma dividida pela quantidade de acessos. Sensível a valores extremos.
p75 / p95 (percentis)
p95 = 95% dos acessos carregaram nesse tempo ou menos. Descreve a cauda (os acessos mais demorados) melhor que a média.
% ≥ 5s / % ≥ 10s
Proporção de acessos acima desses tempos. Os limites são configuráveis por cliente.
Distribuição por tempo de carregamento
Como os acessos se dividem entre faixas de tempo (ex.: < 3s, 3–5s, 5–10s, ≥ 10s).

Observações

  • Amostragem: apenas uma fração dos page views é registrada (taxa atual: —, configurável por cliente). Os percentuais e percentis representam o todo; o card "Page views no período" mostra a amostra e a estimativa de page views reais (amostra ÷ taxa), além da velocidade estimada por hora e por minuto.
  • Retenção: só as últimas 3 horas ficam no banco; o mais antigo é apagado a cada ciclo de agregação.
  • Dados agregados: atualizam periodicamente (não são page view a page view). O horário de atualização e a janela aparecem no topo.
  • Datas e horários no fuso de São Paulo.
