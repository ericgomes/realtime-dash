Sobre este dashboard e métricas

Monitor ao vivo da velocidade de carregamento com usuários reais, coletado via GTM. RUM = Real User Monitoring (Monitoramento de Usuários Reais): mede a performance nos navegadores de quem realmente acessa o site, não em testes simulados. Cada page view é um carregamento de página. Mostra apenas as últimas 3 horas (o histórico fica no GA4) e apresenta os dados de forma neutra: tempos e distribuições, sem julgar.

Métricas de tempo

TTFB
Resposta do servidor (primeiro byte) — indica rede/backend.
DOM Ready
Estrutura da página pronta, antes de imagens pesadas.
Load
Página totalmente carregada (o tempo "página pronta").
p95
95% dos page views carregaram nesse tempo ou menos — enxerga os piores casos melhor que a média.
Faixas de tempo
Como os page views se dividem (< 1s … ≥ 10s). Os limites das faixas são configuráveis por cliente.

Como ler os números

  • Amostragem: registramos ~— dos page views. Os percentuais e o p95 valem para o todo; os cards de volume mostram a amostra e estimam o total real (amostra ÷ taxa).
  • Volume: as tabelas priorizam as páginas mais acessadas. Páginas com menos de 3 page views não entram; use o "mínimo de page views" para focar no que tem tráfego.
  • Bots: tráfego de robôs (Googlebot e outros, detectados pelo User-Agent) é excluído de todos os números; o card "% bots" mostra quanto foi filtrado.
  • Dados agregados a cada ~1 min · horários no fuso de São Paulo.
